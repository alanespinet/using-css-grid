CSS Grid - Columns

The grid-template-columns property

Columns in CSS Grid can be represented by different properties, being grid-template-columns the most common one. This property will accept one or more values representing:

  • The number of columns of the Grid
  • The width of each one

As we will see in a bit, it's important to note that there is a difference between real columns number and visible columns number. The first one represents the amount of columns the grid is featuring, while the second one represents the amount of columns we are seeing.

Column widths in CSS Grid can be represented by absolute values, like pixels and relative values like percentages and a new unit inherent to Grid: fractions.

Examples

Common Measurements

1- We create a grid with 3 columns, each one of 100px width

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: 100px 100px 100px;
}

Where each of the 100px appearances relates to one of the columns. If we color the columns a bit (with the grid-container having a gray background color), we could see the result:

We can see that, since we are giving each column an absolute width (100px), they wont fill the whole containing space. All the gray area to the right is the containing DIV

Also note that, by default, the height of the grid elements is set to stretch. This is one of the best things we have in CSS Grid.

The columns can have, also, different widths, of course:

NOTE: Except when updated, HTML for an example will be always the last one shown.

2- We create a grid with 3 columns, the first one with 100px width, the second one with 200px width and the third one with 150px width:

.grid-container {
    display: grid;
    grid-template-columns: 100px 200px 150px;
}

We can also use percentages. These values will be responsive to the container's width:

3- We create a grid with 3 columns, the first one with 15% of its container's width, the second one with 30% of its container's width and the third one again with 15% of its container's width:

.grid-container {
    display: grid;
    grid-template-columns: 15% 30% 15%;
}

If we want to fill the gap, we can use the special value auto:

4- We create a grid with 3 columns, the first and third ones with 15% of its container's width, and the second one with the remaining available space, since it uses auto as its width value:

.grid-container {
    display: grid;
    grid-template-columns: 15% auto 15%;
}

As it is obvious, we can mix values:

5- We create a grid with 4 columns, the first one with 100px of width, the second one with 25% of its container's width, the third one with 250px of width and the last one with the rest of the space, since it uses auto as its width value:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: 100px 25% 250px auto;
}

Fraction Measurements

Fraction Measurements, represented as fr, determine a "fraction" of the container's width. A "fraction" will always be relative to the total number of columns in the Grid. This means that, for example, in a 4-column grid, 2fr will mean half of the space, while 1fr will mean 25% of the container's width. It is not common to combine "fractions" with other measurements.

For example:

6- We create a grid with 3 columns total. The first one and third one have 1fr, that is 25% of the container's width, while the second one has 2fr, that is 50% of the container's width:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
}

"Fractions" are a relative measurement, what it means that they will "expand and shrink" depending on the container's width.

7- Sometimes you need to create several columns of the same width. For example, in the following example, you want to create 5 columns with 1fr each, that is every column will use 20% of the container's width. You can do it like this:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr 1fr 1fr;
}

The repeat function

8- You can also use the special repeat function. This function receives the amount of columns we want to create and the width each one of them will have. This way, 1fr 1fr 1fr 1fr 1fr would be also represented as repeat(5, 1fr):

.grid-container {
    display: grid;
    grid-template-columns: repeat(5, 1fr);
}

9- Interestinly enough, when you create columns using CSS Grid, if you have more columns in your HTML than the ones defined in the CSS, the extra columns will be arranged in the "next available row":

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
    <div class="column-6"></div>
    <div class="column-7"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(5, 1fr);
}

We will go into this in Rows, but you can see how rows are taking also the vertical space of the container

10- We can use repeat (or several fr measurements): to create homogeneous tables very simply with grid:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
    <div class="column-6"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
}

Once again, because of the nature of Grid, grid cells are stretch. You will see later that you can control this.

The minmax function

11- Since fr "shares" the width of the container based of the coeficient of the units and the amount of columns, we can find ourselves in a situation where a particular columns "shrinked" more than what we needed. For cases like these ones, we can use the special minmax function. This function can be used in in place of a measurement of a column, and receives the minimun and maximum values that column can have. For example, we might have 5 columns equally spaced (1fr each) but maybe, we dont want the middle one to be smaller than 500px. We can do it like this:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: 1fr 1fr minmax(500px, 1fr) 1fr 1fr;
}

If the column is greater than 500px, it will use 1fr as its measurement. If, however, the column were to shrink below 500px, the minmax function would keep it at 500px, because that is its minimim width.Since we have 2 columns with 1fr to each side of the minmax call, we could've also done this, using repeat:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(2, 1fr) minmax(500px, 1fr) repeat(2, 1fr);
}

auto-fill and auto-fit

12- Another one of these especial values is auto-fill. We use it by replacing the "number" in repeat, and its effect will be auto-adding (and auto-removing) columns from a row as long as we have (or not) a specific space (columns and moved up and down between rows). For example, in the following code, we create 300px width columns in every row as long as we have the space available for them:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
    <div class="column-6"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fill, 300px);
}

NOTE: Resize the browser and check how columns cells are moved to the first and second rows depending on the available space. That is the effect of auto-fill.

13- One PRETTY IMPORTANT detail about auto-fill is that it creates empty columns if we have the space to contain them but not the elements. For example, in the following code, we keep the CSS shown in the previous example, but we delete the columns 3 to 6 from the HTML:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fill, 300px);
}

You can see that only two columns are shown. But the inspector shows a VERY different story:

14- As we can see, auto-fill will "create" more columns if we don't have enough elements. That doesnt happen with our new special value: auto-fit. The use of it it's exactly the same:

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, 300px);
}

You can see now that we don't have the auto-columns. Columns will be created ONLY if elements exist:

15- One really cool way of using auto-fit is combining it with minmax to get all the available space unless there are new elements to place. We need two examples for showing this:

First with three elements: All the space is used

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
}

And now with five elements: All the space is used and elements are created in row 2

<div class="grid-container">
    <div class="column-1"></div>
    <div class="column-2"></div>
    <div class="column-3"></div>
    <div class="column-4"></div>
    <div class="column-5"></div>
</div>

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
}

Go ahead and resize the browser window (shrink it) to see how, in both examples, whenever there is no remaining space to hold enough 300px elements per row, the grid changes reorganizing the cells to fit in new rows. This technique is one of those ones we use a lot to keep beautiful table-like grids "stable".

NOTE: This example doesn't mean that auto-fill is never used just because auto-fit exists. Sometimes we DO NEED the extra columns created by auto-fill.
